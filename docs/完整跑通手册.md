# 企业内部知识与文档 RAG Copilot 完整跑通手册

## 一、你现在需要准备什么

这个项目已经具备完整链路，最少只需要下面这些资料和环境：

### 必要环境
- Python 3.11
- Node.js 18+
- 已安装项目依赖

### 可选环境
- Ollama
- 本地模型：`qwen2.5:7b`、`deepseek-r1:7b`、`llama3.1:8b`

### 不装 Ollama 也能演示吗
可以。

如果你没有本地模型，直接使用 `mock` provider，也能完整演示：
- 登录
- 文档接入
- 索引构建
- 检索问答
- 历史记录

## 二、我已经给你准备好的资料

项目里已经可以生成一套可直接进入知识库的中文资料，分为两类：

### 1. 本地演示资料
生成后会放到：
- `backend/data/demo_zh/`

包含：
- 项目概览
- 后端接口说明
- RAG 工作流说明
- 模块职责说明
- 变更说明
- 运维检查表
- 演示问题清单

### 2. 中国官方技术资料
生成后会放到：
- `backend/data/demo_zh_web/`

来源是中国官方技术站点：
- 阿里云帮助中心
- 腾讯云文档
- 华为云文档

说明：
- 这些文档由脚本联网抓取并清洗后保存。
- 抓取失败不会影响本地演示资料生成。

## 三、如何生成资料

在项目根目录执行：

```powershell
cd backend
python scripts/prepare_cn_demo_data.py
```

如果你只想先生成本地资料，不联网抓取：

```powershell
cd backend
python scripts/prepare_cn_demo_data.py --skip-web
```

生成完成后，你会看到：
- `backend/data/demo_zh/`
- `backend/data/demo_zh_web/`
- `backend/data/demo_manifest.json`

## 四、完整跑通步骤

### 1. 启动后端

```powershell
cd backend
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### 2. 启动前端

```powershell
cd frontend
npm install
npm run dev
```

### 3. 登录后台

默认账号：
- 用户名：`admin`
- 密码：使用你在 `.env` 中配置的 `RAG_COPILOT_INITIAL_ADMIN_PASSWORD`

### 4. 准备知识库资料

先执行：

```powershell
cd backend
python scripts/prepare_cn_demo_data.py
```

### 5. 重建索引

进入前端后台：
- 打开“文档管理”
- 点击“重新构建索引”

或者直接调用接口：

```bash
curl -X POST http://127.0.0.1:8000/api/documents/reindex \
  -H "Authorization: Bearer YOUR_TOKEN"
```

### 6. 查看索引状态

```bash
curl http://127.0.0.1:8000/api/index/status \
  -H "Authorization: Bearer YOUR_TOKEN"
```

如果 `vector_count > 0`，说明索引已经可用。

### 7. 发起问答

推荐先选 `mock`：

```bash
curl -X POST http://127.0.0.1:8000/api/chat \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d "{\"query\":\"RAG 工作流是怎样的？\",\"provider\":\"mock\",\"top_k\":4}"
```

### 8. 查看历史记录

进入前端后台：
- 打开“历史记录”

或者调用接口：

```bash
curl http://127.0.0.1:8000/api/history \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## 六、如果你想接入真实本地模型

### 安装 Ollama 后拉取模型

```bash
ollama pull qwen2.5:7b
ollama pull deepseek-r1:7b
ollama pull llama3.1:8b
```

### 问答时选择 provider
- `qwen`
- `deepseek`
- `llama`

如果模型不可用，系统会自动回退到 `mock`。

## 七、最稳妥的演示建议

如果你要确保现场演示 100% 跑通，建议这样做：

1. 先用 `python scripts/prepare_cn_demo_data.py` 生成资料。
2. 先在“文档管理”执行一次“重新构建索引”。
3. 问答时优先选择 `mock`。
4. 只有在你本地已经确认 Ollama 模型可用时，再切换到 `qwen`、`deepseek` 或 `llama`。

这样可以把“环境不稳定”带来的风险降到最低。
